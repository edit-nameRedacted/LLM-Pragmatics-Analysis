## Regenerating hidden-state NPZ files

The files `data/rdm/hidden_states_*.npz` are excluded from git because they are 63–150 MB each (total ~870 MB). They must be regenerated by running the RSA data-collection pipeline:

```bash
python scripts/model_prompting/qxc_main_RSA.py --qwen
python scripts/model_prompting/qxc_main_RSA.py --llama
python scripts/model_prompting/qxc_main_RSA.py --mistral
python scripts/model_prompting/qxc_main_RSA.py --deepseek_v2_lite
# DeepSeek-V3 (if available):
python scripts/model_prompting/qxc_main_RSA.py --deepseek
```

Output NPZ files land in `data/rdm/` alongside their companion `hidden_states_meta_*.csv` files. The RDM notebook (`notebooks/rdm_analysis_colab_fixed_again.ipynb`) expects both in that location.

## Environment and authentication

Running any script in `scripts/model_prompting/` requires access to gated HuggingFace models. You must provide your own HuggingFace access token. The pipeline reads it from the `HF_TOKEN` environment variable — set it in whichever way suits your setup:

```bash
# Option A: shell session
export HF_TOKEN=hf_...

# Option B: .env file (gitignored, loaded automatically if python-dotenv is installed)
echo "HF_TOKEN=hf_..." > .env
```

The token needs at least read access to the model repos used in the experiment (Qwen2.5-7B-Instruct, Llama-3.1-8B-Instruct, Mistral-7B-Instruct, DeepSeek-V2-Lite-Chat). Make sure you have accepted the licence agreements on the HuggingFace model pages before running.
